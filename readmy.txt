1) Отдельное консольное приложение для заполнения БД данными валютных курсов за 2017 и 2018 год (колонка Rate в отчётах CNB).
src/RateReader.Console/RateReader.Console.csproj
Configuration
YearsToHandle[] список лет импортируемых из сервиса (2017, 2018)
HandledPackageSize технический параметр для сохранения полученной инфомации пакетами. Указывается количество записей в пакете.

2) Задача по расписанию, которая сохраняет текущий курс в БД. Период запуска задаётся в конфигурации.
src/RateReader.Scheduler/RateReader.Scheduler.csproj
Configuration
TimerPeriodSec - период чтения данных в секундах

3) Реализовать web API, с помощью которого можно получить отчет по курсу кроны в заданный месяц/год.
src/RateReader.Api/RateReader.Api.csproj
Configuration
ApplicationParameters.CurrenciesCodes - список кодов валют для которых будет строится отчетность

Запуск задачи по рассписанию и API приложения осуществляется через docker. Запуск консоли через exe или vs.
Для работы необходимо 

1. В файл hosts прописать 
127.0.0.1 currency-db	currency-api.local

2. Прописать необходимые параметры в appsettings.json приложений

3. Развернуть хранилище БД (контейнер создается при запуске docker приолжения в проекте).
Накатить миграции из проекта  src\RateReader.Data

Swagger интерфей API приложения доступен по ссылке http://currency-api.local/swagger/index.html
Параметр format принимает значения "txt" или "json". Параметр Month не обязятелен.
